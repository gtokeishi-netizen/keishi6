# 助成金投稿 × Google Spreadsheet 完全連携システム

## 🎯 概要

この連携システムにより、Google スプレッドシートとWordPressの助成金投稿を完全に同期できます。

### ✅ 実現される機能

- **双方向同期**: スプレッドシート ↔ WordPress の両方向でデータ同期
- **リアルタイム更新**: 変更は5分以内に自動同期
- **完全連携**: 削除・公開状態の変更も即座に反映
- **フィールド完全対応**: すべてのカスタムフィールドとタクソノミーを同期
- **エラーハンドリング**: 失敗時の自動リトライとエラー通知

## 📋 セットアップ手順

### Step 1: WordPress側の準備

#### 1.1 アプリケーションパスワードの作成

1. WordPress管理画面にログイン
2. **ユーザー > プロフィール** に移動
3. 下部の「アプリケーションパスワード」セクションで新しいパスワードを作成
4. アプリケーション名: `Google Sheets Sync`
5. 生成されたパスワードをコピー（後で使用）

#### 1.2 REST APIの有効化確認

以下のURLにアクセスして、APIが動作することを確認：
```
https://joseikin-insight.com/wp-json/wp/v2/grant
```

### Step 2: Google Spreadsheet の作成

#### 2.1 新しいスプレッドシートを作成

1. [Google Sheets](https://sheets.google.com) にアクセス
2. 新しいスプレッドシートを作成
3. シート名を `grant_import` に変更

#### 2.2 Google Apps Script の設定

1. スプレッドシートで **拡張機能 > Apps Script** をクリック
2. デフォルトのコードを削除
3. [gas-sync-code.js](./gas-sync-code.js) の内容をすべてコピー&ペースト
4. ファイルを保存（Ctrl+S）

#### 2.3 スクリプトプロパティの設定

1. Apps Script エディタで **設定（歯車アイコン）** をクリック
2. **スクリプト プロパティ** タブを選択
3. 以下のプロパティを追加：

| プロパティ | 値 |
|-----------|---|
| `WP_SITE_URL` | `https://joseikin-insight.com` |
| `WP_USERNAME` | `keishiadmin` |
| `WP_APP_PASSWORD` | （Step 1.1で作成したパスワード） |

#### 2.4 権限の許可

1. Apps Script エディタで **実行 > 関数を実行 > setupSync** を選択
2. 権限の確認画面で「許可」をクリック
3. Google アカウントを選択し、すべての権限を許可

### Step 3: 初期セットアップの実行

#### 3.1 自動セットアップ

1. スプレッドシートに戻る
2. メニューバーに「WordPress連携」が追加されていることを確認
3. **WordPress連携 > 初期セットアップ** をクリック
4. セットアップ完了のメッセージが表示されるまで待つ

#### 3.2 手動セットアップ（必要な場合）

自動セットアップが失敗した場合：

1. Apps Script エディタで以下の順序で関数を実行：
   - `setupSync()`
   - `checkConfiguration()`
   - `manualSync()`

### Step 4: 動作確認

#### 4.1 WordPress → スプレッドシート同期の確認

1. WordPress管理画面で新しい助成金投稿を作成・公開
2. スプレッドシートを確認（5分以内に反映）
3. データが正しく同期されていることを確認

#### 4.2 スプレッドシート → WordPress同期の確認

1. スプレッドシートで新しい行を追加
2. 必要なフィールドに入力
3. 「ステータス」列に `publish` と入力
4. 5分以内にWordPressで投稿が作成されることを確認

## ⚙️ 運用方法

### 日常的な更新

#### 方法1: スプレッドシートから更新

1. スプレッドシートの該当行を編集
2. 「同期状態」列が自動的に `pending` に変更
3. 5分以内にWordPressに反映

#### 方法2: WordPressから更新

1. WordPress管理画面で投稿を編集・更新
2. 5分以内にスプレッドシートに反映

### 投稿の削除

#### スプレッドシートから削除

1. 「ステータス」列を `trash` または `削除` に変更
2. 自動同期でWordPressからも削除される
3. スプレッドシートの行も自動削除

#### WordPressから削除

1. WordPress管理画面で投稿を削除
2. スプレッドシートからも自動で行が削除

### 公開状態の変更

「ステータス」列の値を変更するだけで反映：

- `draft` → 下書き
- `publish` → 公開
- `private` → 非公開
- `trash` → ゴミ箱

## 🔧 トラブルシューティング

### よくあるエラーと対処法

#### 1. 「権限がありません」エラー

**原因**: アプリケーションパスワードが正しくない

**対処法**:
1. WordPressで新しいアプリケーションパスワードを作成
2. Apps Script のスクリプトプロパティを更新
3. `checkConfiguration()` を実行して確認

#### 2. 同期が動作しない

**原因**: トリガーが設定されていない

**対処法**:
1. **WordPress連携 > 自動同期開始** をクリック
2. または Apps Script で `setupAutoSyncTrigger()` を実行

#### 3. データが部分的にしか同期されない

**原因**: 列の位置がずれている

**対処法**:
1. ヘッダー行の順序を確認
2. 必要に応じて `initializeSheet()` を再実行

#### 4. エラーが頻発する

**確認方法**:
1. **WordPress連携 > 設定確認** をクリック
2. Apps Script のログを確認（表示 > 実行履歴）

### デバッグ機能

#### 手動同期の実行

```javascript
// Apps Script エディタで実行
manualSync()
```

#### 同期状態の確認

```javascript
// Apps Script エディタで実行
checkConfiguration()
```

#### ログの確認

## 🚀 高度な設定

### 同期間隔の変更

`CONFIG.SYNC_INTERVAL_MINUTES` を変更：

```javascript
const CONFIG = {
  SYNC_INTERVAL_MINUTES: 5, // 5分間隔
  // ...
};
```

### バッチサイズの調整

大量データの処理時は `BATCH_SIZE` を調整：

```javascript
const CONFIG = {
  BATCH_SIZE: 50, // 一度に処理する件数
  // ...
};
```

### エラー通知の設定

メール通知を有効にする場合は `sendErrorNotification()` を修正：

```javascript
function sendErrorNotification(error) {
  MailApp.sendEmail({
    to: 'admin@joseikin-insight.com',
    subject: '同期エラー通知',
    body: `同期でエラーが発生しました: ${error.message}`
  });
}
```
